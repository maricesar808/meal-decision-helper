# Meal Decision Helper Proposal

## What I'm building

I am building a Meal Decision Helper that helps me choose where to eat by comparing different restaurant or meal options.

## Who it's for / why: name a real person or yourself; what problem does it solve?

This tool is for myself. When I go out to eat with friends or family, I am really indecisive and have a hard time choosing a place. This tool solves that problem by making the decision easier, faster, and more straightforward.

## The state it tracks: what data does the tool need to remember?

The tool will track a list of restaurant or meal options. For each option, it will remember the name, cost level, healthiness rating, and travel time.

## Core features: (3-5 things it will do)

1. Add different restaurants or meal options.
2. Rate each option based on cost, healthiness, and travel time.
3. Calculate the best overall choice.

## What I don't know yet: which JavaScript concepts will you need to figure out?

I need to figure out how to store the meal options, calculate a score for each option, update the page when the user adds or changes something, and reset the tool using JavaScript.
